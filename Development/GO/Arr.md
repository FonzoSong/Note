## 一、Go语言数组的基本概念

Go语言中的**数组**是一种固定长度的序列，用于存储同类型的数据。数组在内存中是连续分配的，通过索引可以访问和修改数组中的元素。

### 数组的核心特点：
- **固定长度**：数组的长度在声明时就确定，无法在运行时改变
- **元素类型一致**：数组中的所有元素必须是相同类型
- **内存连续**：数组在内存中是连续存储的，访问速度非常快

## 二、数组的声明与初始化

### 1. 声明方式
```go
// 显式定义长度和类型
var arr [5]int

// 初始化时定义长度
arr := [5]int{1, 2, 3, 4, 5}

// 使用省略号自动计算长度
arr := [...]int{1, 2, 3, 4, 5}
```

### 2. 访问和修改元素
```go
arr := [5]int{1, 2, 3, 4, 5}
fmt.Println(arr[0])  // 输出: 1
arr[2] = 10          // 修改第三个元素
```

### 3. 遍历数组
```go
// 使用for循环
for i := 0; i < len(arr); i++ {
    fmt.Println(arr[i])
}

// 使用range
for index, value := range arr {
    fmt.Printf("Index: %d, Value: %d\n", index, value)
}
```

## 三、数组与切片的核心区别

| 特性         | 数组                 | 切片                         |
| ------------ | -------------------- | ---------------------------- |
| **长度**     | 固定                 | 动态                         |
| **类型**     | 值类型               | 引用类型                     |
| **内存**     | 连续分配             | 包含指针、长度、容量的结构体 |
| **函数传递** | 传递副本             | 传递引用（共享底层数组）     |
| **扩容**     | 无法扩容             | 可以动态扩容                 |
| **使用场景** | 长度已知且固定的数据 | 需要动态扩展的数据           |

### 关键区别示例：
```go
// 数组作为函数参数传递（传递副本）
func modifyArray(arr [3]int) {
    arr[0] = 100
}
numbers := [3]int{1, 2, 3}
modifyArray(numbers)
fmt.Println(numbers) // 输出: [1 2 3] (未改变)

// 切片作为函数参数传递（传递引用）
func modifySlice(slice []int) {
    slice[0] = 100
}
slice := []int{1, 2, 3}
modifySlice(slice)
fmt.Println(slice) // 输出: [100 2 3] (已改变)
```

## 四、长度与容量的详细解释

### 1. 数组的长度
- 数组的长度在声明时确定，无法改变
- 通过`len(arr)`获取
- 示例：`arr := [5]int{1,2,3,4,5}`，长度为5

### 2. 切片的长度与容量
- **长度（Length）**：切片中当前实际包含的元素个数
  - 通过`len(slice)`获取
  - 可以通过`append`等操作动态改变

- **容量（Capacity）**：从切片起始位置到底层数组末尾的元素个数
  - 通过`cap(slice)`获取
  - 切片可以安全扩展到容量范围，超过容量会触发扩容

### 3. 切片长度与容量的关系
```go
a := [5]int{1, 2, 3, 4, 5}
s := a[1:3] // s: [2, 3]
fmt.Println(len(s)) // 输出: 2 (长度)
fmt.Println(cap(s)) // 输出: 4 (容量)
```
- 切片`s`的长度=2（元素2和3）
- 切片`s`的容量=4（从索引1开始到底层数组末尾有4个元素：2,3,4,5）

### 4. 切片扩容规则
- 当切片长度达到容量上限时，切片会进行扩容
- 扩容策略（Go 1.16+）：
  - 如果原长度<1024，新容量=原容量×2
  - 如果原长度≥1024，新容量=原容量×1.25

## 五、数组与切片的实际应用

### 数组的典型应用场景
1. **固定长度数据集合**：如一周7天的温度数据
   ```go
   var weekTemperatures [7]float64
   ```
   
2. **多维数组**：如矩阵、图像处理
   ```go
   var matrix [3][3]int
   ```

3. **性能要求高的场景**：如嵌入式系统中的固定长度传感器数据

### 切片的典型应用场景
1. **动态数组**：处理需要动态扩展的数据
   ```go
   slice := make([]int, 0, 5) // 初始长度0，容量5
   ```

2. **数据分片**：对大数据进行分片处理
   ```go
   // 将大数组分成多个小切片
   for i := 0; i < len(bigArray); i += chunkSize {
       chunk := bigArray[i:min(i+chunkSize, len(bigArray))]
       // 处理chunk
   }
   ```

3. **共享数据**：多个函数共享同一个底层数组
   ```go
   baseArray := [5]int{1, 2, 3, 4, 5}
   slice1 := baseArray[0:2]
   slice2 := baseArray[2:4]
   // slice1和slice2共享baseArray
   ```

## 六、重要注意事项

1. **数组长度不可变**：
   ```go
   arr := [3]int{1, 2, 3}
   // arr = append(arr, 4) // 编译错误：first argument to append must be slice
   ```

2. **切片的容量管理**：
   - 合理设置初始容量，减少扩容次数
   - 避免切片操作超出长度范围
   - 注意多个切片共享底层数组时的数据一致性

3. **切片与数组的转换**：
   ```go
   // 数组转切片
   arr := [5]int{1, 2, 3, 4, 5}
   slice := arr[:]
   
   // 切片转数组（仅当切片长度等于数组长度时）
   slice := []int{1, 2, 3, 4, 5}
   var arr [5]int
   copy(arr[:], slice)
   ```

## 七、总结

- **数组**：固定长度、类型一致、内存连续，适合长度已知且固定的场景
- **切片**：动态长度、引用类型、可以扩容，是Go语言中更常用的数据结构
- **长度**：表示当前实际包含的元素个数
- **容量**：表示从切片起始位置到底层数组末尾的元素个数

在Go语言开发中，**除非明确需要固定长度，否则优先使用切片**，因为切片提供了更高的灵活性，能更好地处理动态数据。理解长度和容量的区别对于高效使用切片至关重要，合理设置初始容量可以减少内存重新分配的次数，提高程序性能。

> **Tip**：在使用切片时，记得通过`make([]T, length, capacity)`显式设置初始容量，避免频繁扩容带来的性能开销。
